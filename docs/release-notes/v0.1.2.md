# Local Context Bridge v0.1.2

User-experience patch focused on preventing intermittent white-screen hangs and blank console windows during desktop operations.

## Downloads

| Asset | Use this when | Contents |
| --- | --- | --- |
| `Local-Context-Bridge-v0.1.2-Windows-Portable.zip` | You want the complete portable package | Desktop executable, README, license, and this release note. |
| `Local-Context-Bridge-v0.1.2-Windows-x64-setup.exe` | You want a normal Windows setup wizard | NSIS installer. |
| `Local-Context-Bridge-v0.1.2-Windows-x64.msi` | You want a Windows Installer package | MSI installer. |

## Improved

- Moves heavy Tauri commands to blocking worker threads so the WebView stays responsive while scanning, analyzing, previewing, applying, verifying, rolling back, and closing TRAE.
- Runs Windows PowerShell helper processes with `CREATE_NO_WINDOW`, preventing blank console windows during app discovery and TRAE process shutdown.
- Runs `explorer` path opening through the same hidden-process helper on Windows.
- Adds `-NonInteractive` to PowerShell calls used by the TRAE provider.

## Notes

The app still performs real local database work, so long operations can take a few seconds on large databases. The difference is that the frontend should continue painting its busy state instead of becoming white or showing Windows "not responding" in the title bar.
