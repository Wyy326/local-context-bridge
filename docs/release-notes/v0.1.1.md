# Local Context Bridge v0.1.1

Patch release for a WAL checkpoint bug that could make preview/apply fail with `patched database failed integrity_check`.

## Fixed

- Correctly applies SQLite WAL frames only through the last committed transaction boundary.
- Truncates the database copy to the committed `dbsize` from the WAL instead of leaving stale high-numbered pages in the snapshot.
- Ignores frames after the last commit, so uncommitted WAL tail data is not copied into the plaintext analysis database.
- Adds a source snapshot integrity check before migration SQL runs, so failures are reported as preparation issues instead of being misattributed to project/session transfer logic.

## Why This Matters

The previous WAL copy logic could overlay every frame from `database.db-wal` and leave the snapshot larger than the actual committed database. On some TRAE databases this produced SQLite btree errors such as duplicate page references, invalid page numbers, and `never used` pages during `pragma integrity_check`.

This release fixes the snapshot preparation stage. It does not restore deleted sessions and does not change the transfer model.
